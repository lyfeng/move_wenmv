## 📄 WEN MOVED Coin ($WENMO) 水龙头界面需求文档

### I. 界面目标与设计原则

* **目标：** 提供一个**简单、清晰、一键式**的界面，让用户能够轻松连接钱包并领取每日限额的 $WENMO$ 代币。
* **设计原则：** **极简主义 (Minimalist)**，突出核心功能——**领取 (Claim)**。
* **主题：** 应与 **Movement** 项目的主题（安全、速度、MoveVM）和 **$WENMO$ Meme** 的形象（例如，带向上箭头的卡通形象）保持一致。

---

### II. 核心功能需求

| 模块 | 需求描述 | 技术实现要点 |
| :--- | :--- | :--- |
| **1. 钱包连接** | 界面顶部必须有一个明显的 **"Connect Wallet"** 按钮。 | 使用 **Wallet Adaptors** 或 **Web3Modal** 来支持主流钱包（如 MetaMask、Move-Compatible 钱包等）。|
| **2. 领取状态显示** | 钱包连接后，界面应立即显示以下三种状态之一：| 需要调用合约的**读取函数 (View Function)** 或查询链上存储。|
| | **A. ✅ 可领取：** 用户已冷却，可立即领取。 |
| | **B. ⏳ 冷却中：** 显示剩余冷却时间（倒计时）。 | 使用 JavaScript 的 `Date` 对象计算当前时间与上次领取时间的时间差。|
| | **C. ❌ 已领完：** 如果水龙头余额不足（低优先级，仅作为提醒）。 | 持续查询水龙头模块的 $WENMO$ 余额。|
| **3. 核心领取按钮** | 界面中央应有一个大且醒目的 **"CLAIM 100 $WENMO$"** 按钮。 | |
| | **功能：** 点击按钮后，调用 Move 合约中 `WENMO_FAUCET` 模块的 `claim_wenmo` **入口函数 (Entry Function)**。| 必须处理交易签名、提交到 M2 链，并处理交易成功或失败（如冷却失败）的提示。|

---

### III. 界面信息展示需求

| 区域 | 显示内容 |
| :--- | :--- |
| **顶部标题** | **WEN MOVED Coin ($WENMO) Faucet** |
| **钱包信息** | 已连接的钱包地址（部分隐藏）、网络名称（Movement M2/M3）。 |
| **代币总量** | **水龙头剩余余额：** 显示水龙头资源中剩余的 $WENMO$ 数量。 |
| **个人信息** | **您的 $WENMO$ 余额：** 显示当前连接钱包中持有的 $WENMO$ 数量。 |
| **提示信息** | 简短的说明，例如：“每日可领取 100 $WENMO$，领取间隔 24 小时。” |

---

### IV. 技术实现与部署要求

1.  **前端技术栈：** HTML5, CSS3, JavaScript (可使用 React/Vue 等框架加速开发)。
2.  **Web3 库：** `ethers.js` 或 `web3.js`，用于处理钱包连接和合约交互。
3.  **链交互适配：** 由于 Move 语言和 MoveVM 的特殊性，需要确保 Web3 库能够通过 M2 链的 **EVM-兼容 RPC** 与 Move 合约的 **EVM 代理/包装器**进行交互，或者直接使用 Movement Labs 官方提供的 **SDK/API** 进行 Move 原生函数的调用。
4.  **部署：** 部署到 **Vercel/Netlify** 或 **IPFS**（如前所述），以获取一个稳定的 URL 供用户访问。
